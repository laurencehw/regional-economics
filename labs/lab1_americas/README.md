# Lab 1 - Americas

## Objective
Estimate spatial spillovers in regional GDP growth across the Americas using a trade-weighted spatial lag model, and quantify how border frictions shift estimated propagation effects.

## Core Method
Spatial Lag Model (SAR) with trade-weighted W matrix

## Primary Datasets
- WDI (macro outcomes and controls)
- UN Comtrade (bilateral trade weights)
- BTS and customs/border indicators (friction proxies)

## Variable Mapping (Wired)
- `gdp_growth` <- WDI indicator `NY.GDP.MKTP.KD.ZG`
- `log_gdp_pc` <- log of WDI indicator `NY.GDP.PCAP.KD`
- `manufacturing_share` <- WDI indicator `NV.IND.MANF.ZS`
- `border_delay_index` <- BTS `border_delay_index` column
- Trade weights for W <- Comtrade `trade_value_usd` aggregated by reporter-partner-year

Mappings are configured in `data/source_mappings.json` and transformed via `code/prepare_lab1_inputs.py`.
Comtrade pulls can be generated by `code/fetch_comtrade_api.py` using `COMTRADE_API_KEY` from env or `C:\Users\lwils\.claude\settings.json`.

## Research Question
How much of a region's GDP growth is explained by neighbor growth transmitted through trade linkages, after controlling for local fundamentals and border frictions?

## Folder Layout
- `data/`: raw and mapped lab data
- `code/`: scripts
- `output/`: tables, figures, and model results

## Build Checklist
1. Harmonize source IDs and map raw fields into canonical variables.
2. Construct and row-standardize bilateral trade matrix W.
3. Build cross-section by selected year.
4. Estimate SAR and compare to non-spatial baseline.
5. Run robustness checks on alternative W definitions and friction variables.
6. Export reproducible outputs and interpretation notes.

## Minimum Deliverables
- Baseline model summary (`output/model_summary.json`).
- Spatial weight matrix artifact (`output/weight_matrix.csv`).
- Estimation-ready cross-section (`output/cross_section_used.csv`).
- One short memo interpreting rho and policy relevance.
